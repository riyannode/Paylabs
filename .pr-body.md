## Summary

Add standalone Vercel backend for Paylabs — a Chrome extension + API that gates content with live x402/Gateway nanopayments on Arc.

### What's included

**Infrastructure:**
- Hono app with Vercel serverless entry (`api/index.ts` → `vercel.json` rewrite)
- Standalone `package.json` (no monorepo cross-imports)
- Centralized config from env vars
- postgres.js client with connection pooling
- Full DB schema (8 tables + seed data for arc-community + sepiasearch)

**Routes (10 route files):**
- `GET /health` — implemented
- `POST /api/auth/*` — nonce, verify, me (stubs)
- `GET|POST /api/gateway/*` — status, deposit, sync (stubs)
- `POST /api/content/*` — resolve, quote, unlock (stubs)
- `GET /api/access/check` (stub)
- `POST /api/x402/*` — requirements, verify, settle (stubs)
- `GET /api/receipts/*` (stubs)
- `GET|POST /api/settlement/*` (stubs)
- `GET /api/sites/*` (stubs)
- `POST /api/ai/*` — request, execute, :id (stubs)

**Services (10 service files — core logic implemented):**
- `x402Gateway.ts` — EIP-3009 requirements builder, Arc USDC domain
- `settlementBatch.ts` — batch threshold=5, auto-close, receipt creation
- `accessPass.ts` — create/check access passes after payment
- `receipts.ts` — query receipts with batch status + txHash
- `siteValidation.ts` — Arc Community + SepiaSearch URL validation
- `contentId.ts` — deterministic content ID from URL hash
- `tavily.ts` — web search for AI Discover
- `openaiSummary.ts` — source ranking + explanation
- `idempotency.ts` — duplicate payment prevention
- `security.ts` — origin validation + wallet address sanitization

### Vercel deployment

```
Root Directory: apps/vercel-backend
Framework: Other
Install: npm install
Build: npm run build
```

### Payment model

- $0.000001 USDC per content unlock
- Real x402/EIP-3009 authorization (no fake payments)
- Gateway batched settlement on Arc Testnet
- Settlement batch threshold: 5 payments → batch closes
- Real txHash from Gateway/Arc (never faked)
- Explorer: `https://testnet.arcscan.app/tx/{txHash}`

### CORS allowed origins

- `chrome-extension://*`
- `community.arc.io`
- `community.arc.network`
- `sepiasearch.org`
