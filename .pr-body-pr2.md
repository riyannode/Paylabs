## feat(auth): SIWE wallet auth + paid-action model cleanup — PR #2

Branch: `feat/auth-paid-action-model`

### What changed

**Auth (SIWE + JWT)**
- `POST /api/auth/nonce` — generates EIP-4361 nonce (`crypto.randomBytes`, 5 min TTL, single-use)
- `POST /api/auth/verify` — verifies SIWE signature via viem `verifySiweMessage`, returns JWT
- `GET /api/auth/me` — returns authenticated user (Bearer JWT required)
- SIWE domain check (host from `PAYLABS_PUBLIC_ORIGIN`) + chainId check (`5042002`)
- JWT signed with HMAC-SHA256, configurable expiry via `JWT_EXPIRES_IN` (default 24h)
- No dev bypass, no fake signature path, no x402 payment triggered by auth

**Naming cleanup**
- `content_access` → `thread_open` in x402Gateway purpose routing
- `/api/content/unlock` → `/api/content/open-thread`
- `CONTENT_OPEN_PRICE_USDC` → `THREAD_OPEN_PRICE_USDC` (with env fallback)
- `AGENT_ALLOW_CONTENT_ACCESS` → `AGENT_ALLOW_THREAD_OPEN` (with env fallback)
- DB schema CHECK: added `'thread_open'` alongside `'content_access'` for backward compat

**Root endpoint**
- Updated purpose description to reflect new product direction

### New files
- `src/services/auth.ts` — nonce gen, SIWE verify, JWT sign/verify, user lookup
- `src/middleware/jwt.ts` — Hono middleware for Bearer JWT extraction
